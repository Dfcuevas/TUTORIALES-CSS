Combinadores

Los ultimos selectores que veremos son los llamados selectores de combinacion. Se llaman asi porque combinan otros selectores de manera que proporciona una relacion util entre ellos y la ubicacion del contenido en el documento.

Selector de descendientes

Ya hemos visto los selectores de descendientes en articulos anteriores (selectores con espacios entre ellos).

body article p

Estos selectores seleccionan elementos que son descendientes de otros elementos. No es necesario que sean hijos directos.

Selector de combinacion de elementos hijo.

El selector de combinacion de elementos hijo es un simbolo de (mayor que) (>), que selecciona solo cuando los selectores identifican elementos que son hijos directos. Los elementos descendientes que se encuentran al final de la jerarquia, no se seleccionan. Por ejemplo, para seleccionar solo los elementos p que son hijos directos de elementos article

article > p

Hermanos adyacentes

El selector de elementos hermanos adyacentes (+) se utiliza para seleccionar un elemento que se encuentre justo al lado de otro elemento en el mismo nivel de jerarquia. Por ejemplo, para seleccionar todos los elementos img que aparecen justo despues de elementos p:

p + img

El caso de uso mas comun es modificar el parrafo que va justo despues del titulo.. si ponemos algun elemento entre h1 y p, el estilo ya no se aplicaria a la regla, ya que no p no estaria inmediatamente despues del h1.

Hermanos generales

Si deseas seleccionar los hermanos de un elemento, incluso si no son directamente adyacentes, puedes utilizar el combinador de hermanos general (~). Por ejemplo, para seleccionar todos los elementos <img> que aparecen despues de los elementos p, hacemos esto:

p ~ img

Este combinador funciona incluso si hay algun elemento incrustado entre los elementos referenciados en la regla. Todos los elementos img que vayan despues de p, son seleccionados, si estan en el mismo nivel de jerarquia no importanto si hay elementos distintos entre ellos.
